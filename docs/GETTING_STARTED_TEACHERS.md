## Creating Your Teacher Account

### Sign Up

1. Go to pi-cast homepage
2. Click "Sign Up as Teacher"
3. Choose your sign-up method:
   - **Email**: Enter email and verify with code
   - **GitHub**: Connect your GitHub account
4. Complete registration

### Request Teacher Status

If you signed up as a student:
1. Go to **Settings** → **Account**
2. Click "Request Teacher Status"
3. Fill out the teacher application form
4. Wait for admin approval (usually 24-48 hours)

## Creating Your First Lesson

### Lesson Planning

Before you start creating:

1. **Define Learning Objectives**
   - What should students learn?
   - What prerequisites are needed?
   - What's the expected outcome?

2. **Outline Your Content**
   - Break into logical sections
   - Plan equation demonstrations
   - Prepare example problems

3. **Gather Materials**
   - Equations you'll use
   - Example problems
   - Any supplementary content

### Creating a Lesson

1. Click **Create** → **New Lesson**
2. Fill in lesson details:
   - **Title**: Clear and descriptive
   - **Description**: What students will learn
   - **Category**: Select appropriate category
   - **Level**: Beginner, Intermediate, or Advanced
   - **Tags**: Add relevant keywords

3. Click **Create Lesson**

### Building Lesson Content

#### Adding Equation Blocks

1. Click **+ Equation** in the toolbar
2. Type your equation (e.g., `y = mx + c`)
3. The equation is tokenized with syntax highlighting
4. Position the block on the canvas

#### Adding Control Blocks

1. Click **+ Controls** in the toolbar
2. Select variables to expose
3. Set min/max ranges for sliders
4. Position controls near related equations

#### Adding Chart Blocks

1. Click **+ Chart** in the toolbar
2. Select equations to plot
3. Configure axes and scaling
4. Position chart for optimal viewing

#### Adding Description Blocks

1. Click **+ Description** in the toolbar
2. Add explanatory text
3. Use markdown for formatting
4. Include LaTeX for math notation

### Arranging Your Canvas

- **Drag blocks** to reposition
- Blocks **auto-arrange** to avoid overlap
- Use the **grid** for alignment (toggle with Show Grid)
- Group related equations together

## Recording Lessons

### Preparing to Record

1. **Test Your Setup**
   - Check microphone quality
   - Test audio levels
   - Ensure quiet environment

2. **Prepare Your Script**
   - Outline key points
   - Plan transitions between topics
   - Prepare examples

3. **Do a Test Recording**
   - Record a short test segment
   - Check audio quality
   - Verify block positions

### Starting Recording

1. Open your lesson in edit mode
2. Arrange all blocks in starting positions
3. Click the **Record** button (red circle)
4. Wait for the countdown
5. Begin teaching!

### During Recording

**Recording Controls:**
- 🎤 **Microphone**: Mute/unmute
- ⏸️ **Pause**: Pause recording temporarily
- 🔖 **Bookmark**: Mark important moments
- ⏹️ **Stop**: End recording

**Best Practices:**
- Speak clearly and at moderate pace
- Reference blocks as you discuss them
- Use bookmarks for key concepts
- Pause between major topics

### Audio Segmentation

pi-cast automatically:
- Detects silence in your audio
- Segments speech from silence
- Synchronizes audio with events
- Creates clean playback experience

### Re-recording Sections

If you make a mistake:
1. Pause the recording
2. Note the timestamp
3. Continue from a good point
4. Edit during post-processing (future feature)

## Publishing and Sharing

### Reviewing Your Recording

Before publishing:
1. Play back the full lesson
2. Check audio quality throughout
3. Verify all events synchronized
4. Review bookmarks are accurate

### Setting Visibility

Choose your lesson visibility:
- **Private**: Only you can view
- **Unlisted**: Anyone with link can view
- **Organization**: Members of your organization
- **Public**: Visible to all users

### Publishing

1. Click **Publish** button
2. Fill in final details:
   - Thumbnail image
   - Estimated duration
   - Prerequisites
3. Submit for review (if required)
4. Lesson goes live after approval

### Sharing Your Lesson

Share via:
- **Direct Link**: Copy lesson URL
- **Social Media**: Share to Twitter, Facebook
- **Embed**: Embed in your website (future)
- **Organization**: Share within organization

## Tips for Great Lessons

### Content Quality

1. **Start with Objectives**
   - Clearly state learning goals
   - Reference throughout lesson
   - Summarize at the end

2. **Use Visual Examples**
   - Show, don't just tell
   - Use multiple representations
   - Connect algebra to graphs

3. **Pace Appropriately**
   - Not too fast, not too slow
   - Pause for key concepts
   - Allow time for experimentation

4. **Build Progressively**
   - Start with basics
   - Add complexity gradually
   - Connect to prior knowledge

### Technical Quality

1. **Audio Quality**
   - Use a good microphone
   - Record in quiet space
   - Speak clearly and consistently

2. **Visual Clarity**
   - Use large, readable fonts
   - High contrast colors
   - Organized canvas layout

3. **Interaction Design**
   - Place controls intuitively
   - Group related elements
   - Leave space for graphs

### Engagement Strategies

1. **Ask Questions**
   - Pose problems before solving
   - Encourage prediction
   - Check understanding

2. **Use Bookmarks**
   - Mark key concepts
   - Create navigation points
   - Help students review

3. **Provide Practice**
   - Include example problems
   - Suggest exercises
   - Link to related lessons
